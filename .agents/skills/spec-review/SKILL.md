# Specification Review

## Trigger

- PLAN変更
- 要件変更
- ユーザー指示変更

## Procedure

1. PLANとの差分を確認する

2. 不足情報を質問する

3. SPECを更新する

4. 必要なら再度質問する

5. SPECが完成するまで繰り返す

6. 人間へレビューを依頼する

7. 承認されるまで更新する

8. 承認後にTODOを見直す

9. 影響範囲を整理する

## Rules

- 曖昧な仕様では実行しない
- 人間との認識合わせを優先する
- SPECが承認されるまでTODOを更新しない
- 画像を使うように指示された場合はどの画像を使うか詳細まで詰める
